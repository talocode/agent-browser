# Agent Browser v0.2.0 — Persistent Sessions & Trace Reports

## Summary

Agent Browser v0.2.0 adds persistent logical sessions, step tracing, and structured JSON/Markdown reports for multi-step agent browser workflows.

## Highlights

- **Sessions** — create, list, and close local agent sessions
- **Trace** — every session-aware action records evidence (status, warnings, console/network counts, screenshots)
- **Reports** — export JSON or Markdown summaries with recommended next actions
- **CLI** — `session create|list|close|trace|report` plus `--session` on browser commands
- **MCP** — five new `browser_session_*` tools; optional `sessionId` on existing tools

## Install

```bash
npm install -g @talocode/agent-browser@0.2.0
npx playwright install chromium
```

## CLI examples

```bash
agent-browser session create --name deploy-check --json
agent-browser check https://example.com --session <sessionId> --screenshot-out ./deploy.png --json
agent-browser console --session <sessionId>
agent-browser network --session <sessionId>
agent-browser session report <sessionId> --format markdown
agent-browser session close <sessionId>
```

## MCP tools added

- `browser_session_create`
- `browser_session_list`
- `browser_session_close`
- `browser_session_trace`
- `browser_session_report`

Existing `browser_navigate`, `browser_snapshot`, `browser_screenshot`, `browser_console`, `browser_network`, and `browser_check` accept optional `sessionId`.

## Safety

- Unsafe protocols and private network targets blocked by default
- Localhost only with `AGENT_BROWSER_ALLOW_LOCALHOST=1`
- Sensitive query params redacted in network evidence
- No credential automation, CAPTCHA bypass, or anti-detection

## Limitation

v0.2 persists **logical session metadata and trace evidence**, not live Playwright page state across separate CLI invocations. Each CLI call may launch a fresh browser while still recording steps against the same session.
